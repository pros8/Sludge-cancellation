# Влияние сладжа на отмену цифровых подписок

Данное приложение разработано на фреймворке oTree для проведения лабораторного эксперимента. Проект направлен на изолированное измерение влияния деструктивной архитектуры выбора (сладжа) на поведение потребителей при расторжении цифровых контрактов.

## Дизайн эксперимента
Участники выполняют монотонную когнитивную задачу (подсчет нулей в матрицах 10х10) в условиях убывающего вознаграждения (штраф 1 ЭВЕ/сек). В фоновом режиме на учетную запись действует подписка на «Премиум-ассистента», которая автоматически списывает баллы начиная с 3-го раунда. 

### Межгрупповой дизайн (Between-subjects)
* **Контрольная группа (Прозрачный дизайн):** Отмена подписки в личном кабинете реализуется в 2 клика с моментальным подтверждением.
* **Тестовая группа (Сладж):** Отмена усложнена 4-шаговым модальным окном. Используются элементы когнитивного давления: манипулятивные тексты, обязательный опрос, маскировка интерфейса и чувствительность к регистру и скрытым символам при вводе кодового слова.

## Рандомизация
Рандомизация реализована на уровне сессии в функции `creating_session`. Используется итератор `itertools.cycle([True, False])`, гарантирующий идеальное разделение участников в пропорции 1:1. 

## Собираемые метрики
Для эконометрического анализа (OLS, Логит-регрессии) и непараметрических тестов в базу данных экспортируются следующие поведенческие переменные:
* `cancelled_subscription` — факт успешной отмены (1/0).
* `cancel_round` — раунд успешной отмены подписки.
* `cabinet_time_round` — трансакционные издержки (время в секундах, проведенное на страницах отмены).
* `time_to_first_click_ms` — время до первого клика в миллисекундах (прокси когнитивного ступора).
* `clicks_count` — количество хаотичных кликов внутри модального окна.
* `validation_errors` — количество ошибок при вводе кодового слова.

## Экономические стимулы
* **Валюта:** Экспериментальные валютные единицы (ЭВЕ). Курс конвертации: 10 ЭВЕ = 1 рубль.
* **Базовая выплата:** Гарантированные 300 рублей за участие.

## Структура проекта
Основной код эксперимента изолирован в приложении `sludge_cancellation`. Ключевые файлы:

* **`sludge_cancellation/__init__.py`** — серверная архитектура oTree. Содержит параметры эксперимента (`Constants`), модели данных базы (`Player`), логику рандомизации групп и методы расчета итоговых выплат.
* **`sludge_cancellation/Task.html`** — интерфейс основной задачи (генерация матриц) и механика Личного кабинета. Вся логика модальных окон, таймеров убывающего вознаграждения, фиксации времени до первого клика и хаотичной навигации реализована на JavaScript.
* **`sludge_cancellation/Instructions.html`, `Survey.html`, `Results.html`** — шаблоны вспомогательных экранов (инструкция, финальная анкета и расчет итоговой выплаты).
* **`settings.py`** (в корне проекта) — конфигурация сессии oTree, регистрация переменных для сохранения между раундами (`PARTICIPANT_FIELDS`) и настройка курса конвертации ЭВЕ в рубли.

## Инструкция по запуску
1. Клонируйте репозиторий и установите зависимости: `pip install -r requirements.txt`
2. Запустите локальный сервер: `otree devserver`
3. Для запуска эксперимента на реальной выборке перейдите в панель администратора (вкладка Sessions -> Create new session), выберите конфигурацию `sludge_cancellation` и укажите требуемое количество участников.
